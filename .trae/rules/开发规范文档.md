# 前端代码开发规范 v1.0（面向AI代码生成Agent）

## 1. 文档目标

- **确保生成的前端代码**：语义正确、可访问、高性能、安全、易维护。
- **适用技术栈**：HTML5、CSS（含预处理器：SCSS/Less）、JavaScript (ES2020+)、TypeScript、React、Vue（3.x）、Angular（可选）。
- **Agent行动原则**：优先遵循现代Web标准与最佳实践，避免过时或危险模式。

---

## 2. 通用规范（语言无关）

### 2.1 文件组织
- 一个文件对应一个组件/模块（React: `MyComponent.jsx`，Vue: `MyComponent.vue`）。
- 样式文件与组件同目录或位于专用`styles/`文件夹，使用`.module.css`或CSS-in-JS（项目约定）。
- 资源文件（图片、字体）使用相对路径，并放置在`assets/`目录。

### 2.2 命名规范
| 类型 | 规则 | 示例 |
|------|------|------|
| HTML文件 | 全小写，短横线分隔 | `user-profile.html` |
| CSS类名 | 小写，短横线分隔（BEM可选） | `.btn-primary`, `.card__title` |
| JavaScript变量/函数 | camelCase | `fetchUserData`, `isLoading` |
| TypeScript类型/接口 | PascalCase | `UserProps`, `ApiResponse` |
| 常量 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |
| React组件 | PascalCase | `UserProfile`, `Button` |
| Vue组件 | PascalCase 或 kebab-case（模板中） | `UserProfile` / `<user-profile>` |
| 自定义Hooks | `use` 前缀 + camelCase | `useWindowSize` |
| 事件处理函数 | `handle` 前缀 | `handleClick`, `handleSubmit` |

### 2.3 缩进与换行
- 缩进：2个空格（HTML/CSS/JS/TS/Vue/JSX）。
- 行宽：最大100字符。
- 文件末尾保留一个空行。

## 3. HTML规范

### 3.1 文档结构
- 使用`<!DOCTYPE html>`声明，`<html>`元素应设置`lang`属性。
- 字符编码：`<meta charset="UTF-8">` 必须出现在`<head>`最前面。
- viewport设置：`<meta name="viewport" content="width=device-width, initial-scale=1.0">`（移动优先）。

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>页面标题</title>
    ...
</head>
<body>
    ...
</body>
</html>
```

### 3.2 语义化
- 根据内容含义使用正确的元素：`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`。
- 避免无意义的`<div>`和`<span>`堆砌；使用`<ul>`/`<ol>`表现列表，`<button>`用于可点击操作，`<a>`用于导航链接。
- 为交互元素提供明确的`:focus`样式（键盘可访问）。

### 3.3 属性规范
- 使用双引号包裹属性值：`<div class="container">`。
- 布尔属性可以不写值（如`disabled`、`checked`、`required`）。
- `alt`属性必须出现在`<img>`上（无装饰性图片时使用空`alt=""`）。
- `title`属性仅在`iframe`或需要额外说明时使用，不应替代`alt`。

## 4. CSS规范

### 4.1 选择器与特异性
- 避免使用ID选择器（`#id`），因其特异性过高。优先使用类选择器。
- 避免嵌套过深：Sass/Less嵌套不超过3层。
- 不使用`!important`，除非用于强制覆盖第三方样式或工具类（如显示隐藏）。

### 4.2 命名与组织
- 推荐BEM（Block Element Modifier）命名：
  ```css
  .card { }              /* Block */
  .card__title { }       /* Element */
  .card--large { }       /* Modifier */
  ```
- 公用样式放在`global.css`或`variables.css`中，定义CSS自定义属性（变量）：
  ```css
  :root {
      --primary-color: #0066cc;
      --spacing-unit: 8px;
  }
  ```
- 使用`@import`仅用于导入第三方库或抽象文件（变量、混合），避免大量串联。

### 4.3 响应式与布局
- 默认移动优先（Mobile First）：先编写小屏样式，再通过`min-width`媒体查询增强。
- 使用Flexbox或Grid进行布局，避免传统浮动布局（除非用于文字环绕）。
- 尺寸单位：`rem`用于字体、间距（基准16px），`px`用于边框、阴影等物理属性，`%`/`vw`/`vh`用于容器尺寸。
- 断点建议：
  ```css
  /* 手机 */       0 - 639px
  /* 平板竖屏 */   640px - 1023px
  /* 桌面 */       1024px - 1439px
  /* 宽屏 */       1440px +
  ```

### 4.4 性能与可维护性
- 避免使用`*`通用选择器重置所有边距（可改用现代reset如`normalize.css`或极简reset）。
- 尽量减少CSS文件体积：合并重复规则，使用简写属性（`margin: 0 auto`）。
- 动画使用`transform`和`opacity`（只触发复合图层），避免改变`width`/`top`/`left`等触发重排的属性。
- 对于预处理器（SCSS），合理使用`@mixin`、`@extend`，注意`@extend`可能导致冗长选择器。

## 5. JavaScript/TypeScript规范

### 5.1 语言特性
- 使用ES2020+语法：`let`/`const`、箭头函数、模板字符串、解构、展开运算符、可选链`?.`、空值合并`??`。
- 不使用`var`；不使用`==`（始终使用`===`和`!==`）。
- 使用`async/await`代替原始Promise回调（除非需要`Promise.all`等并发）。
- 模块化：使用`import`/`export`代替`require`。

### 5.2 变量作用域与类型（TypeScript）
- 优先使用`const`，仅在需要重新赋值时使用`let`。
- TypeScript项目必须定义接口/类型，禁止滥用`any`。使用`unknown`代替`any`，使用类型守卫收窄类型。
- 函数参数和返回值必须有类型注解。

```typescript
interface ButtonProps {
    label: string;
    onClick: () => void;
    disabled?: boolean;
}
```

### 5.3 函数设计
- 单一职责：一个函数只做一件事，长度不超过30行。
- 默认参数放在末尾：`function fetchData(url: string, timeout: number = 5000)`
- 避免副作用（直接修改外部对象），尽量保持纯函数。

### 5.4 DOM操作
- 优先使用`querySelector`/`querySelectorAll`，避免使用`getElementsByClassName`（返回动态集合）。
- 不直接使用`innerHTML`拼接用户内容（XSS风险）。使用`textContent`设置纯文本，使用`createElement` + `appendChild`或安全的框架模板。
- 事件监听：在组件卸载时必须移除监听器（`removeEventListener`或框架生命周期）。避免内存泄漏。

### 5.5 异步错误处理
- 所有`async`函数调用必须包裹`try/catch`或使用`.catch()`，且给用户适当反馈（如toast提示）。
- 对于`fetch`，必须检查`response.ok`（因为网络错误才会进入catch，HTTP状态码200-299才视为成功）。

```javascript
async function fetchData() {
    try {
        const res = await fetch('/api/data');
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const data = await res.json();
        // 处理数据
    } catch (err) {
        console.error('获取数据失败', err);
        showToast('加载失败，请稍后重试');
    }
}
```

### 5.6 性能模式
- 避免在循环中读取/修改DOM或进行高开销计算；使用`documentFragment`批量更新。
- 使用防抖（debounce）和节流（throttle）优化高频事件（滚动、输入、resize）。
- 频繁更新的UI（动画）推荐使用`requestAnimationFrame`。

## 6. 框架特定规范（React & Vue）

### 6.1 React (Hooks + 函数组件)

#### 组件结构顺序（推荐）：
1. 导入（第三方 → 相对路径）
2. 类型定义（TypeScript interface）
3. 组件声明
4. Hooks（useState, useEffect, useCallback, useMemo等）
5. 事件处理函数
6. 渲染辅助函数 (renderXxx)
7. JSX return

#### Hooks规则：
- 只在顶层调用Hooks（不在循环/条件中）。
- 自定义Hook必须以`use`开头。
- `useEffect`依赖项必须完整声明，避免空依赖数组导致的闭包陷阱。
- 使用`useCallback`/`useMemo`仅当传递给子组件或作为其他hook依赖时，避免过度优化。

#### 状态管理：
- 简单UI状态用`useState`；跨组件状态使用Context或状态管理库（Zustand/Redux Toolkit）。
- 避免将整个store作为依赖；使用选择器提取最小状态。

#### JSX规范：
- 自闭合标签：`<img src="..." />`、`<MyComponent />`。
- 多行属性每个属性单独一行（>2个属性）。
- 条件渲染：`{isLoading && <Spinner />}` 或三元表达式，避免复杂逻辑内嵌。
- 列表渲染必须提供稳定的`key`（不推荐用索引，除非列表静态且无重排）。

### 6.2 Vue 3 (Composition API + `<script setup>`)

#### 组件结构：
```vue
<script setup lang="ts">
// 导入
// props / emits 定义
// 响应式变量 (ref, reactive)
// 计算属性
// 侦听器
// 生命周期钩子
// 方法
</script>

<template>
  <!-- 模板 -->
</template>

<style scoped>
/* 作用域样式 */
</style>
```

#### 规范细则：
- Props使用TypeScript定义：`defineProps<{ title: string }>()`。
- 使用`defineEmits`声明事件，避免直接修改props。
- 模板中避免复杂表达式，提取到计算属性。
- `v-for`必须与`:key`一起使用，key优先使用唯一标识。
- `v-if`和`v-for`不要同时用于同一元素（用`<template>`包装）。

### 6.3 通用（React/Vue公共）
- 组件文件大小建议 ≤ 300行；超过时应拆分。
- 不要在组件内直接定义内联函数作为子组件props，除非函数非常轻量且子组件经过memo化（但默认建议用`useCallback`或提取到外部）。
- 避免prop drilling，使用上下文/注入。
- 所有组件的`props`必须验证类型（TypeScript或PropTypes）。

## 7. 可访问性（A11y）规范（MUST）

- 所有交互元素必须可通过键盘访问（tabindex自然顺序，无需人为设置>0）。
- 为交互元素提供可见的`:focus-visible`样式（不要移除outline除非提供替代样式）。
- 为视频/音频提供字幕或文字说明（`<track>`）。
- 使用`aria-label`或`aria-labelledby`为没有文本标签的按钮/图标提供名称。
- 避免仅通过颜色传递信息（例如“红色表示错误”），需补充文字或图标。
- 页面必须有`<h1>`，且标题层级正确（h1-h6不跳级）。
- 对于动态内容更新（表单错误、弹窗），使用`aria-live`区域。

```html
<button aria-label="关闭" @click="close">×</button>
<div aria-live="polite" role="status">{{ message }}</div>
```

## 8. 安全规范（前端特有）

| 风险 | 禁止 | 要求 |
|------|------|------|
| XSS | 使用`innerHTML`、`document.write`、`eval()` | 使用`textContent`、`innerText`；若必须使用HTML，则使用DOMPurify进行净化 |
| 跨站请求伪造 | 未使用CSRF Token的API请求 | 对于状态变更请求（POST/PUT/DELETE），使用CSRF令牌（通常在meta标签或cookie中） |
| 敏感数据暴露 | 将token、用户凭证存储在前端localStorage/sessionStorage（风险中等） | 推荐使用httpOnly cookie存储认证凭据；若必须用localStorage，确保内容非敏感且与XSS防护结合 |
| 第三方脚本 | 直接加载不可信CDN脚本 | 使用SRI（子资源完整性）哈希校验：`<script src="..." integrity="sha384-..." crossorigin="anonymous">` |
| URL伪协议 | `javascript:void(0)` | 使用`<button>`或处理`click`事件并调用`preventDefault` |

- 所有用户输入在插入DOM之前必须转义。框架默认是安全的（React/Vue转义），但不要绕过（如`v-html`，`dangerouslySetInnerHTML`）。
- 使用`Content-Security-Policy`头部（生成HTML时建议添加）。
- 对于链接，验证外部链接的`rel="noopener noreferrer"`。

## 9. 性能与加载优化

- **资源加载**：
  - 图片使用`<img loading="lazy">`（懒加载）或现代`<picture>`元素。
  - 字体使用`font-display: swap`避免白屏。
  - JavaScript模块使用动态导入（`import()`）进行代码分割。
- **关键渲染路径**：
  - 同步脚本放在`</body>`前或使用`defer`/`async`属性。
  - CSS内联关键样式（首屏样式），非关键样式异步加载。
- **避免布局抖动**：为图片/视频设置`width`/`height`属性或CSS宽高比（`aspect-ratio`）。
- **资源体积**：
  - 图片压缩，使用webp格式（提供fallback）。
  - CSS/JS开启压缩（minify），移除console.log（生产环境）。

## 10. 测试与示例

Agent生成的前端代码**应包含**单元测试和/或组件测试用例：

- 测试框架：Jest + Testing Library（React/Vue）或Vitest。
- 每个组件至少有：
  - 渲染测试（快照可选，尽量使用断言）。
  - 事件交互测试（点击、表单提交）。
  - 边界条件（props为空、无效值）。
- 测试命名：`it('should render button with label')`。

**示例（React + Testing Library）**：
```jsx
import { render, screen, fireEvent } from '@testing-library/react';
import Button from './Button';

test('点击按钮时调用onClick', () => {
    const handleClick = jest.fn();
    render(<Button label="提交" onClick={handleClick} />);
    fireEvent.click(screen.getByText('提交'));
    expect(handleClick).toHaveBeenCalledTimes(1);
});
```

## 13. 示例：完全符合规范的前端组件

**React + TypeScript + CSS Module**：

```tsx
// Button.module.css
.btn {
    display: inline-flex;
    align-items: center;
    padding: 8px 16px;
    border: none;
    border-radius: 4px;
    background-color: var(--primary-color);
    color: white;
    cursor: pointer;
    transition: background 0.2s ease;
}
.btn:focus-visible {
    outline: 2px solid #000;
    outline-offset: 2px;
}
.btn:disabled {
    opacity: 0.6;
    cursor: not-allowed;
}
// Button.tsx
import React from 'react';
import styles from './Button.module.css';

interface ButtonProps {
    label: string;
    onClick: () => void;
    disabled?: boolean;
    ariaLabel?: string;
}

export const Button: React.FC<ButtonProps> = ({
    label,
    onClick,
    disabled = false,
    ariaLabel,
}) => {
    return (
        <button
            className={styles.btn}
            onClick={onClick}
            disabled={disabled}
            aria-label={ariaLabel || label}
        >
            {label}
        </button>
    );
};

// Button.test.tsx
import { render, screen, fireEvent } from '@testing-library/react';
import { Button } from './Button';

describe('Button', () => {
    it('renders label and responds to click', () => {
        const mockFn = jest.fn();
        render(<Button label="Click me" onClick={mockFn} />);
        const button = screen.getByRole('button', { name: 'Click me' });
        fireEvent.click(button);
        expect(mockFn).toHaveBeenCalledTimes(1);
    });
});
```
